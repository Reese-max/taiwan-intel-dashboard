## 5. 實作結果（2026-06-19，已完成並驗證）

設計修正：經查核既有程式碼，專案早已用「統計 → `IntelEvent`」模式處理彙總資料
（`fetchFraudDashboard`、`fetchCrimeWeekly`、`miaoliStatEvents`）。因此**不另建面板**，
三個新來源一律比照 `fetchFraudDashboard` emit 單筆 `IntelEvent`，事件自動流入既有
清單/地圖/時間軸 UI，前端零改動。

新鮮度更正：原先憑 metadata 的下載 URL 參數（`ym=11101`）誤判 103351「stale 至 2022」。
實查資料證明其已更新至 **115年（2026）4月**，並未過時。已實際抽列驗證後採用。

| 來源 | dataset | 實作 fetcher | 最新期別 | 實跑結果 |
|---|---|---|---|---|
| 全國刑案發生率／破獲率 | 103351 + 103352 | `fetchCrimeRate` | 115年4月 | 發生率 220.84 件/10萬人口、破獲率 81.45% |
| 臺中市取締酒駕 | 88170 | `fetchDuiTaichung` | 2026-04 | 取締 389 件（移送法辦 184、拒測 62、肇事 23）|
| 臺北市家暴通報 | 145744 | `fetchDvTaipei` | 114年12月 | 合計 758 件（按案件類型分）|

實作摘要：
- `scripts/lib/twinkle-query.mjs`：`queryTwinkleRows` 向後相容擴充 `columns` / `group_by`（家暴找最新期別用）。
- `scripts/lib/fetch-police.mjs`：新增純 mapper（`mapCrimeRateEvents` / `mapDuiTaichungEvents` /
  `mapDvTaipeiEvents`）+ 期別解析（`parseRocPeriodLabel` / `latestRocPeriod`）+ 3 個 fetch wrapper，
  並接入 `fetchPolice`。
- `scripts/fetch-live.mjs`：`policeSourceDefs` 新增 3 筆 provenance 來源定義（來源健康度可見）。
- `tests/police-stats-mappers.test.ts`：8 個新測試（純 mapper + 期別解析）。

驗證：`npm test` 30/30 綠 · `tsc --noEmit` exit 0 · `npm run build` exit 0 ·
`npm run fetch:police` 三來源 `ok`，警政事件數 +3。
過程修正一個 bug：酒駕「肇事」統計原先誤把「無肇事」一起計入（`includes("肇事")` 連帶命中
「無肇事」），已改為排除無肇事（327 → 23 件）。
